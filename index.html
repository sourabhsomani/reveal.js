DOTA - How Bans Affect Reputation

Bans happen when a player:

  • Abandon matches
  • Gets reported by fellow players

Punishments

  • The more offenses, the longer the ban.
  • Ban time escalates each time a consecutive ban is issued
  • Escalation time resets after an unknown period of good behavior

A Potentinal Solution For Gitcoin

  • Every user has a funder + contributor karma (2 scores)
  • Each bad behavior by the user results in penalization
  • Automated (less weightage) & user initiated

When Does Reputation Drop?

  • When funder/hunter isn't responsive
  • Delay in payments + Scope/Feature creep
  • Work is mediocre

Low Reputation means

  • Less likely their issue will be picked up / issue will be assigned to them.

  • New User / Users with reputation >= 75%
    Sees an Issue -> Start Work -> (Faucet Request) -> Submit -> Get Payout

    Old Users with reputation 45 - 75%
    Sees an Issue -> Can select after ban. Lesser the % longer the ban duration

    Old Users with reputation <= 45%
    Can start work only after ban + will have to deposit a 10% (y$) of the issue bounty(x$) as collateral

If the user has poor reputation and is starts work

  • Happy Path: Successfull completion == y$ + x$
  • Abandonment: x$ can be either sent to gitcoin / the funder (for the inconvience) / can be split btwn the two.
  • 1st makes us look money minded.
    2nd might lead to a situation where funder might game the system by targeting users with low reputation to make money.
  • User will be able to escalate this with a period of N days if they feel they've been done wrong by upon which core team reviews it
